Backend Development Tutorial

Learn how to build powerful server applications that handle data and business logic

What is Backend Development?

Backend development involves building the server-side logic that powers web applications. While frontend is what users see, backend handles data processing, business logic, authentication, and communication with databases. Backend developers build APIs (Application Programming Interfaces) that frontend applications communicate with.

Key Responsibilities:

  • Processing user requests and data
  • Managing authentication and authorization
  • Communicating with databases
  • Building and maintaining APIs
  • Handling business logic and calculations
  • Ensuring security and data protection

How Backend Works - Request/Response Cycle

Backend applications follow a request/response model:

1. Frontend Sends Request

User clicks a button or submits a form. JavaScript sends data to the server.

→

2. Server Receives Request

Backend server receives the request with data and URL parameters.

→

3. Process & Query Database

Backend processes the request, queries database, performs calculations.

→

4. Send Response

Server sends back result (JSON, data, or status message).

→

5. Frontend Uses Response

Frontend updates page based on server response.

Node.js & Express - Practical Example

Node.js allows you to write backend code in JavaScript. Express is the most popular framework for building APIs.

Setting Up a Simple Server

// Import Express
const express = require('express');
const app = express();

// Middleware to parse JSON
app.use(express.json());

// GET endpoint - Retrieve data
app.get('/api/greeting', (req, res) => {
    res.json({ 
        message: 'Hello World!',
        timestamp: new Date()
    });
});

// POST endpoint - Receive data
app.post('/api/users', (req, res) => {
    const user = req.body; // Get data from request
    
    // Process user data
    console.log('New user:', user);
    
    // Send response
    res.json({ 
        success: true,
        userId: 123
    });
});

// Start server on port 3000
app.listen(3000, () => {
    console.log('Server running on port 3000');
});

Different Request Methods (HTTP Verbs)

  • GET - Retrieve data from server
  • POST - Send data to server to create something
  • PUT - Send data to server to update something
  • DELETE - Delete data from server

Building a Complete API

Here's an example of a simple user management API:

const express = require('express');
const app = express();
app.use(express.json());

// In-memory storage (in real app, use database)
let users = [];
let nextId = 1;

// GET all users
app.get('/api/users', (req, res) => {
    res.json(users);
});

// GET specific user
app.get('/api/users/:id', (req, res) => {
    const user = users.find(u => u.id == req.params.id);
    if (!user) {
        return res.status(404).json({ error: 'User not found' });
    }
    res.json(user);
});

// CREATE new user
app.post('/api/users', (req, res) => {
    const { name, email } = req.body;
    
    if (!name || !email) {
        return res.status(400).json({ error: 'Name and email required' });
    }
    
    const user = { id: nextId++, name, email };
    users.push(user);
    res.status(201).json(user);
});

// UPDATE user
app.put('/api/users/:id', (req, res) => {
    const user = users.find(u => u.id == req.params.id);
    if (!user) {
        return res.status(404).json({ error: 'User not found' });
    }
    
    user.name = req.body.name || user.name;
    user.email = req.body.email || user.email;
    res.json(user);
});

// DELETE user
app.delete('/api/users/:id', (req, res) => {
    users = users.filter(u => u.id != req.params.id);
    res.json({ message: 'User deleted' });
});

app.listen(3000, () => {
    console.log('API running on port 3000');
});

Frontend Calling Backend API

Frontend applications use the Fetch API or Axios to communicate with backend servers.

Sending Data to Backend

// GET request - Retrieve data
fetch('http://localhost:3000/api/users')
    .then(response => response.json())
    .then(data => console.log('Users:', data))
    .catch(error => console.error('Error:', error));

// POST request - Send data
const newUser = { name: 'John', email: 'john@example.com' };

fetch('http://localhost:3000/api/users', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(newUser)
})
.then(response => response.json())
.then(data => console.log('User created:', data))
.catch(error => console.error('Error:', error));

// PUT request - Update data
fetch('http://localhost:3000/api/users/1', {
    method: 'PUT',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ name: 'Jane' })
})
.then(response => response.json())
.then(data => console.log('User updated:', data))
.catch(error => console.error('Error:', error));

// DELETE request - Remove data
fetch('http://localhost:3000/api/users/1', {
    method: 'DELETE'
})
.then(response => response.json())
.then(data => console.log('User deleted:', data))
.catch(error => console.error('Error:', error));

Authentication in Backend

Backend must verify that users are who they claim to be before allowing access to protected resources.

JWT (JSON Web Tokens)

const jwt = require('jsonwebtoken');

// Login endpoint - Generate token
app.post('/api/login', (req, res) => {
    const { email, password } = req.body;
    
    // Verify credentials (check against database)
    if (email === 'user@example.com' && password === 'password123') {
        // Create JWT token
        const token = jwt.sign(
            { email: email },
            'your-secret-key',
            { expiresIn: '2h' }
        );
        
        res.json({ token: token });
    } else {
        res.status(401).json({ error: 'Invalid credentials' });
    }
});

// Protected endpoint - Verify token
function authenticateToken(req, res, next) {
    const token = req.headers['authorization'];
    
    if (!token) {
        return res.status(403).json({ error: 'No token provided' });
    }
    
    jwt.verify(token, 'your-secret-key', (err, decoded) => {
        if (err) return res.status(403).json({ error: 'Invalid token' });
        req.user = decoded;
        next();
    });
}

// Use middleware on protected route
app.get('/api/protected', authenticateToken, (req, res) => {
    res.json({ message: 'This is protected', user: req.user });
});

Error Handling Best Practices

Good backend code handles errors gracefully and provides helpful messages.

// Standard error codes:
// 200 - OK (success)
// 201 - Created (resource created)
// 400 - Bad Request (invalid input)
// 401 - Unauthorized (authentication required)
// 403 - Forbidden (permission denied)
// 404 - Not Found
// 500 - Server Error

// Example with proper error handling
app.post('/api/users', (req, res) => {
    try {
        const { name, email } = req.body;
        
        // Validate input
        if (!name || name.trim() === '') {
            return res.status(400).json({ 
                error: 'Name is required' 
            });
        }
        
        if (!email || !email.includes('@')) {
            return res.status(400).json({ 
                error: 'Valid email is required' 
            });
        }
        
        // Create user
        const user = { name, email };
        
        // Success response
        res.status(201).json({ 
            success: true,
            message: 'User created successfully',
            data: user 
        });
        
    } catch (error) {
        console.error('Error creating user:', error);
        res.status(500).json({ 
            error: 'Internal server error' 
        });
    }
});

Backend Best Practices

  • RESTful APIs - Follow REST principles for consistent APIs
  • Input Validation - Always validate data from frontend
  • Security - Hash passwords, use HTTPS, validate tokens
  • Error Handling - Return appropriate HTTP status codes and messages
  • Logging - Log important events for debugging
  • Rate Limiting - Prevent abuse by limiting requests per user
  • Database Optimization - Write efficient queries and use indexing
  • Code Organization - Keep code modular with routes, controllers, models

Getting Started Project

Create your first Node.js API server:

Installation

# Install Node.js from nodejs.org, then:
npm init -y
npm install express
npm install nodemon --save-dev  # Auto-reloads on file changes

Create server.js

const express = require('express');
const app = express();
app.use(express.json());

app.get('/api/test', (req, res) => {
    res.json({ message: 'Backend is working!' });
});

app.post('/api/test', (req, res) => {
    res.json({ 
        received: req.body,
        processedAt: new Date()
    });
});

app.listen(3000, () => {
    console.log('Server running at http://localhost:3000');
});

Run the server

npx nodemon server.js

Test it: Visit http://localhost:3000/api/test in your browser or use Postman!
