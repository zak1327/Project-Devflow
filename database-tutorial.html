Database Tutorial

Learn how to store, organize, and retrieve data effectively using databases

What is a Database?

A database is an organized collection of structured data stored on a server. Instead of keeping data in random files, databases organize information in a structured way using tables with rows and columns, similar to Excel spreadsheets. Databases allow you to quickly search, filter, and retrieve data.

Why Use Databases?

  • Persistence - Data survives even if the server restarts
  • Organization - Data is organized in tables with clear structure
  • Scalability - Handle millions of records efficiently
  • Query Ability - Fast searching and filtering
  • Performance - Optimized retrieval with indexing
  • Security - Access controls and encryption

Types of Databases

SQL / Relational

Structured data in tables with relationships. Most common for business apps.

Examples: MySQL, PostgreSQL, SQL Server

NoSQL / Document

Flexible schema, stores data as documents (JSON). Great for flexible data.

Examples: MongoDB, Firebase, CouchDB

Key-Value

Simple key-value pairs. Fast but limited. Used for caching.

Examples: Redis, Memcached

Graph

Stores data as nodes and relationships. Perfect for social networks.

Examples: Neo4j

SQL (Relational) Databases

SQL (Structured Query Language) databases are the most traditional and widely used. Data is organized in tables with relationships between them.

Understanding Tables

A table is like a spreadsheet with rows and columns:

ID Name Email Score
1 John Smith john@example.com 95
2 Jane Doe jane@example.com 87
3 Bob Wilson bob@example.com 76

Key Concepts

  • Table - Collection of similar data (like users, products)
  • Column - A field/attribute (like name, email, age)
  • Row - A record with data for all columns
  • Primary Key - Unique identifier for each row (usually ID)
  • Foreign Key - References a primary key in another table

SQL Queries - CRUD Operations

CRUD stands for Create, Read, Update, Delete - the basic operations on databases.

CREATE - Insert Data

-- Insert a single record
INSERT INTO users (name, email, score)
VALUES ('John Doe', 'john@example.com', 92);

-- Insert multiple records
INSERT INTO users (name, email, score) VALUES
  ('Jane Smith', 'jane@example.com', 88),
  ('Bob Johnson', 'bob@example.com', 76),
  ('Alice Brown', 'alice@example.com', 95);

READ - Retrieve Data

-- Get all users
SELECT * FROM users;

-- Get specific columns
SELECT name, email FROM users;

-- Filter with WHERE clause
SELECT * FROM users WHERE score > 80;

-- Find specific user
SELECT * FROM users WHERE email = 'john@example.com';

-- Sort results
SELECT * FROM users ORDER BY score DESC;

-- Limit number of results
SELECT * FROM users LIMIT 5;

UPDATE - Modify Data

-- Update a specific user's score
UPDATE users 
SET score = 98 
WHERE email = 'john@example.com';

-- Update multiple fields
UPDATE users 
SET name = 'John Smith', score = 96 
WHERE id = 1;

-- Update multiple records
UPDATE users 
SET score = score + 5 
WHERE score < 80;

DELETE - Remove Data

-- Delete a specific user
DELETE FROM users WHERE email = 'bob@example.com';

-- Delete multiple records
DELETE FROM users WHERE score < 60;

-- Delete all records (use with caution!)
DELETE FROM users;

Advanced SQL Queries

Joining Tables

Combine data from multiple related tables:

-- INNER JOIN - Show only matching records
SELECT users.name, orders.amount 
FROM users 
INNER JOIN orders ON users.id = orders.user_id;

-- LEFT JOIN - Include all from left table
SELECT users.name, COUNT(orders.id) as order_count
FROM users 
LEFT JOIN orders ON users.id = orders.user_id
GROUP BY users.id;

Aggregation Functions

-- Count records
SELECT COUNT(*) as total_users FROM users;

-- Calculate average
SELECT AVG(score) as average_score FROM users;

-- Get maximum value
SELECT MAX(score) as highest_score FROM users;

-- Get minimum value
SELECT MIN(score) as lowest_score FROM users;

-- Sum values
SELECT SUM(score) as total_score FROM users;

-- Group and aggregate
SELECT 
    department, 
    COUNT(*) as employees,
    AVG(salary) as avg_salary
FROM employees
GROUP BY department;

Creating Database and Tables

Create Database

-- Create a new database
CREATE DATABASE my_app_db;

-- Select database
USE my_app_db;

Create Tables

-- Create users table
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    score INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Create orders table with foreign key
CREATE TABLE orders (
    id INT PRIMARY KEY AUTO_INCREMENT,
    user_id INT NOT NULL,
    amount DECIMAL(10, 2),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);

-- Data types:
-- INT - Integer numbers
-- VARCHAR(n) - Text up to n characters
-- DECIMAL(10,2) - Decimal numbers
-- DATE - Date only
-- TIMESTAMP - Date and time
-- BOOLEAN - True/False
-- NOT NULL - Must have a value
-- DEFAULT - Default value
-- AUTO_INCREMENT - Auto generate number
-- UNIQUE - Must be unique in table
-- PRIMARY KEY - Unique identifier
-- FOREIGN KEY - Links to another table

Frontend ↔ Backend ↔ Database Flow

1. User Submits Form

User fills out form and clicks submit in frontend

→

2. Frontend Sends Data

JavaScript sends data to backend API using fetch()

→

3. Backend Receives

Express server receives and validates the data

→

4. Save to Database

Backend executes SQL INSERT query to save data

→

5. Return Response

Backend sends back confirmation (success/error)

→

6. Update Frontend

Frontend updates page to show success message

Example: Saving User Registration

// Frontend - registration-form.html
<form id="regForm">
    <input type="text" id="name" placeholder="Name" required>
    <input type="email" id="email" placeholder="Email" required>
    <button type="submit">Register</button>
</form>

// Frontend - JavaScript
document.getElementById('regForm').addEventListener('submit', async (e) => {
    e.preventDefault();
    
    const data = {
        name: document.getElementById('name').value,
        email: document.getElementById('email').value
    };
    
    // Send to backend API
    const response = await fetch('http://localhost:3000/api/register', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(data)
    });
    
    const result = await response.json();
    if (result.success) {
        alert('Registration successful!');
    }
});

// Backend - Node.js/Express server.js
app.post('/api/register', async (req, res) => {
    const { name, email } = req.body;
    
    try {
        // Insert into database
        const query = 'INSERT INTO users (name, email) VALUES (?, ?)';
        await db.execute(query, [name, email]);
        
        // Return success response
        res.json({ success: true, message: 'User registered!' });
    } catch (error) {
        res.status(500).json({ success: false, error: error.message });
    }
});

Database Best Practices

  • Use Primary Keys - Every table should have a unique identifier
  • Normalize Data - Organize data to reduce redundancy
  • Use Prepared Statements - Prevent SQL injection attacks
  • Index Important Columns - Speed up searches
  • Regular Backups - Never lose data
  • Appropriate Data Types - Use correct types for storage efficiency
  • Validate Input - Check data before inserting
  • Use Transactions - Ensure data consistency
  • Monitor Performance - Track slow queries

Common Database Mistakes to Avoid

❌ SQL Injection

Never directly insert user input into queries

❌ "SELECT * FROM users WHERE id=" + userId

✅ Use Prepared Statements

Always use parameterized queries

✅ "SELECT * FROM users WHERE id=?"

❌ No Backups

Risk losing all data permanently

Automated backups are essential!

✅ Schedule Backups

Automatic daily/weekly backups

Easy recovery if something breaks

Getting Started with MySQL

Installation

# Download MySQL from mysql.com and install

# Start MySQL server
mysql -u root -p

# Create a database
CREATE DATABASE my_app;
USE my_app;

# Create a simple table
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100),
    email VARCHAR(100)
);

# Insert data
INSERT INTO users (name, email) VALUES ('John', 'john@example.com');

# Retrieve data
SELECT * FROM users;

Connect from Node.js

// Install mysql2
npm install mysql2/promise

// In your server.js
const mysql = require('mysql2/promise');

const connection = await mysql.createConnection({
    host: 'localhost',
    user: 'root',
    password: 'your_password',
    database: 'my_app'
});

// Execute query
const [rows] = await connection.execute('SELECT * FROM users');
console.log(rows);
